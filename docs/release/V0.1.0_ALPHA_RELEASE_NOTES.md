# Nous Runtime v0.1.0-alpha

Nous Runtime is a long-lived intelligent runtime for helping people understand, create, and operate complex systems.

## Alpha scope

- Terminal-native conversation with persistent sessions, replay, recovery, slash commands, Dashboard, Inspector, approval controls, and bounded history.
- Server-authoritative Run, Event, Conversation, Approval, Provider, Workflow, Connector, Plugin, Knowledge, Workspace, and Node integration.
- Provider setup for OpenAI-compatible, Anthropic-compatible, Ollama, local HTTP, and custom endpoints using credential references.
- Python, JavaScript, and TypeScript SDK clients; a VS Code MVP; and an experimental Desktop client.
- Governed Workflow, Connector, Plugin, Knowledge, Code Assistant, and Multi-Agent paths.

## Security boundary

The Server Runtime is authoritative. Connector writes and other governed actions pass through Governance and Approval. Plugin and Connector failures are fail-closed. Network services bind to localhost by default. Nous is not an operating-system sandbox; operators remain responsible for host isolation, credentials, TLS, and deployment policy.

## Known limitations

- This is an Alpha pre-release and public compatibility guarantees are limited.
- Desktop and VS Code integrations are MVP surfaces using authenticated HTTP replay rather than WebSocket push.
- Android is Experimental/Unverified, excluded from the public source set, and not release-gating.
- Distributed Node paths are experimental and require explicit pairing and deployment configuration.
- External Provider smoke tests require maintainer-supplied credentials and were not performed with live production accounts.

## Installation

```bash
python -m pip install nous-runtime
nous --version
nous
```
