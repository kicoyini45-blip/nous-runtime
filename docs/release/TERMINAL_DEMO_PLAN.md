# Terminal Demo Plan

The release demo should be a restrained 90-second terminal recording with no credentials or private paths.

1. Open `nous` in a clean temporary workspace and show the compact NOUS header.
2. Type `/` and narrow the palette to `/provider`.
3. Show `/status`, a short conversation, and the separate Runtime activity stream.
4. Open `/runs`, `/run show`, `/dashboard`, and `/inspect`.
5. Demonstrate Ctrl+C cancelling only the active operation, followed by a usable prompt.
6. Restart the client and show restored conversation history and event replay.
7. End with `/quit`.

Use a synthetic workspace, a placeholder Provider name, `NO_COLOR` capture as an accessibility alternative, and a final frame stating `v0.1.0-alpha`. Do not display tokens, user directories, network addresses other than localhost, or internal review material.
